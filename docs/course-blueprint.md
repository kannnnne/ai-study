# 课程蓝图

## 一句话定位

面向银行解决方案研发团队的 AI 编程入门课：用 Claude Code + DeepSeek 从 0 搭建一个最小可运行 Java 微服务 POC，并沉淀一套能复制到真实项目中的 AI 辅助研发工作流。

## 课程目标

### 对学员

- 知道 AI 编程工具能做什么、不能做什么。
- 能独立完成 Claude Code + DeepSeek 的基础配置。
- 能用 AI 辅助完成需求分析、设计评审、任务拆分、开发、测试和验收。
- 能理解主控会话和子会话的分工，避免上下文混乱。
- 能用 AI 辅助搭建一个包含前端、后端、数据库和基础微服务能力的最小 POC。
- 形成基本安全意识：不泄露代码、不盲信输出、不绕过公司安全规范。

### 对团队

- 降低 AI 编程工具的入门门槛。
- 统一一套相对成熟、可复制的使用流程。
- 提升需求澄清、方案设计、代码生成、测试补齐、文档维护和问题排查效率。
- 建立内部 AI 编程实践样板，方便后续扩展到更多工具和场景。

### 对讲师本人

- 借课程倒逼自己系统梳理 AI 编程体系。
- 把“会用工具”升级为“能讲清楚方法论、边界和落地路径”。
- 形成可以持续迭代的内部影响力资产。

## 向领导汇报时的表述

这套课程不是追求讲透大模型底层算法，而是面向当前研发团队最现实的问题：大家知道 AI 有用，但受制于网络、工具、认知门槛和项目安全边界，真正能稳定用起来的人不多。因此课程以 Claude Code + DeepSeek 为主线，用一个从 0 开始搭建的 Java 微服务 POC，沉淀一套能复制、能演示、能在银行项目研发中落地的 AI 编程方法。

课程会覆盖环境搭建、基本概念、需求分析、架构设计、前后端开发、数据库、测试验收、微服务治理和风险边界，让技术同学先从“敢用、会用、用得安全”开始，再逐步扩展到更复杂的 Agent、知识库和自动化研发流程。

## 课程设计原则

- 先解决“能不能跑起来”，再解释“为什么这样做”。
- 先让 POC 最小可运行，再逐步引入微服务复杂度。
- 每节课都要有可复制产出，例如一份需求文档、一个设计方案、一个接口、一个测试或一份验收清单。
- 概念只讲到够用，不在入门阶段展开论文、模型训练和复杂 Agent 框架。
- 对 AI 输出始终强调验证：AI 是副驾驶，不是审批人。

## 推荐课程结构

### 第一部分：认知和环境

- AI 编程到底帮我们解决什么问题。
- 大模型、提示词、上下文、Agent、MCP 等概念的通俗解释。
- Claude Code + DeepSeek 环境搭建。
- 银行项目中的安全边界和使用注意事项。

### 第二部分：从 0 搭建 POC

- 用 AI 做需求分析和验收标准。
- 用 AI 做架构设计和评审。
- 用 AI 搭建 Spring Boot 后端服务。
- 用 AI 设计数据库和测试数据。
- 用 AI 设计前端页面并联调接口。
- 用 AI 从单体演进到微服务。

### 第三部分：形成个人工作流

- 如何写一个好任务描述。
- 如何分阶段让 AI 完成复杂任务。
- 如何把经验沉淀成提示词模板、项目说明和团队规范。
- 常见错误示范：需求太大、上下文太乱、输出不验证、把敏感信息发出去。

## 推荐交付形式

- 视频：每节 15 到 30 分钟。
- 文字讲义：每节 1 到 3 页。
- 操作清单：环境、命令、配置项、注意事项。
- 案例仓库：保留演示代码、提示词、AI 输出和人工修订版本。
- FAQ：持续收集同学问题，后续补录或补文档。

## 边界说明

这不是大模型算法课，也不是 Agent 框架开发课。入门阶段不重点讲模型训练、RAG 系统实现、复杂多 Agent 编排、企业级私有化部署等内容。课程目标是让研发同学先掌握能提升日常工作的 AI 编程方法，并通过一个可运行 POC 建立信心。
