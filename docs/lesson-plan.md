# 分集大纲

## 课程主线

本课程用一个完整的最小可运行 POC 串起来：从需求分析开始，借助 Claude Code + DeepSeek 逐步完成设计、后端、数据库、前端、微服务改造、测试验收和复盘。

推荐总节数：8 到 10 节。

## 第 0 课：课程导入和预期管理

### 视频目标

让学员知道这门课不是单纯介绍 AI 工具，而是用 AI 走完一个小项目的研发闭环。

### 主要内容

- 公司和银行项目研发场景下为什么需要 AI 编程。
- AI 编程不是替代程序员，而是提升理解、生成、检查和整理效率。
- 本课程主线：Claude Code + DeepSeek + Java 微服务 POC。
- 学完后的最低目标：能搭环境、会拆任务、能用 AI 完成一个小型研发闭环。

### 演示建议

展示最终 POC 的效果图或目标结构，让学员知道后面每节课在拼哪块。

## 第 1 课：AI 编程基础概念和工作流

### 视频目标

讲清楚 AI、LLM、提示词、上下文、Agent、工具调用这些关键词，并建立正确工作流。

### 主要内容

- 大模型可以理解为“基于上下文生成结果的通用助手”。
- 提示词不是咒语，而是任务说明、约束和验收标准。
- 上下文窗口决定了 AI 当前能看见什么。
- Agent 可以理解为“会分步骤、会调用工具、会检查结果的 AI 工作流”。
- 一个会话只做一个主线任务。
- 复杂任务必须先需求分析、设计评审、拆分计划、测试验收，再开发。

### 课后材料

- AI 研发工作流图。
- 会话拆分原则。
- 常见错误：一上来就让 AI 写完整系统。

## 第 2 课：Claude Code + DeepSeek 环境搭建

### 视频目标

让同学能在自己的机器上把工具跑起来，并理解网络、安装包和模型访问的区别。

### 主要内容

- macOS、Windows、WSL2 的推荐路径。
- Claude Code 安装。
- DeepSeek API Key 申请和配置。
- `ANTHROPIC_BASE_URL`、`ANTHROPIC_API_KEY` 等环境变量。
- 没有代理时如何判断是安装问题还是运行访问问题。
- 项目级 `CLAUDE.md` 的作用和示例。

### 交付材料

- 安装清单。
- 配置模板。
- 常见错误 FAQ。

## 第 3 课：用 AI 做 POC 需求分析

### 视频目标

演示如何从一个粗略想法开始，让 AI 帮助整理需求、边界和验收标准。

### 演示任务

输入一句粗需求：

```text
我要做一个客户风险评级与账户查询 POC，用于演示银行研发场景下 AI 辅助开发流程。
```

让 AI 输出：

- 用户角色。
- 功能范围。
- 不做什么。
- 业务流程。
- 非功能要求。
- 待确认问题。
- 需求验收标准。

### 课程重点

AI 不应该直接写代码。先让它问问题、定边界、写验收。

## 第 4 课：用 AI 做系统设计和评审

### 视频目标

把需求转成可开发设计，并演示如何让 AI 自己审查设计。

### 演示任务

让 AI 设计：

- 第一阶段单体架构。
- 第二阶段微服务架构。
- 数据库表。
- REST API。
- 统一响应和异常处理。
- 日志和敏感信息规则。
- 测试策略。

### 课程重点

设计不是越复杂越好。入门 POC 要先跑通，再逐步拆微服务。

## 第 5 课：后端最小服务开发

### 视频目标

用 AI 创建并实现第一个 Spring Boot 3.x 后端服务。

### 演示任务

- 创建项目结构。
- 实现客户查询接口。
- 连接数据库。
- 增加参数校验。
- 增加统一返回和异常处理。
- 写单元测试。

### 推荐提示词方向

要求 AI 先分析影响范围、再输出计划、再开发，并在每次修改后说明验证命令。

## 第 6 课：数据库和测试数据

### 视频目标

让学员看到 AI 不只是写业务代码，也能辅助建表、初始化数据和测试数据设计。

### 演示任务

- 设计 customer、account、risk_record、operation_log 表。
- 写 Flyway 或初始化 SQL。
- 构造脱敏样例数据。
- 让 AI 生成边界测试数据。

### 课程重点

银行场景要特别关注金额、日期、状态码和敏感字段。

## 第 7 课：前端页面设计和联调

### 视频目标

展示如何用 frontend-design skill 的思路，让 AI 设计一个专业、克制、可演示的企业后台页面。

### 演示任务

- 客户列表。
- 筛选区域。
- 客户详情抽屉或详情页。
- 风险等级视觉标识。
- 账户信息展示。
- 风险分布图。
- 对接后端接口。

### 课程重点

前端不是让 AI 随便套模板。要先说清用户、场景、视觉方向、组件和验收标准。

## 第 8 课：从单体演进到微服务

### 视频目标

演示如何让 AI 辅助拆分服务，而不是一开始就设计过重。

### 演示任务

- 拆分 customer-service 和 risk-service。
- 引入 gateway-service。
- 接入 Nacos 注册和配置。
- 服务间调用。
- 回归测试。

### 课程重点

拆分前要有理由，拆分后要能验证。AI 可以加速迁移，但架构边界要人把关。

## 第 9 课：AI 验收、人工验收和安全运维验收

### 视频目标

让学员掌握如何检查 AI 产出，而不是只看它能不能生成。

### 演示任务

- 新开干净会话做 AI 审查。
- 按 P0/P1/P2 输出问题。
- 跑测试和构建。
- 手工操作页面。
- 检查日志、密钥、配置、敏感字段。

### 课程重点

真正的完成不是“AI 写完了”，而是需求、设计、测试、安全和人工验收都过了。

## 第 10 课：复盘和个人工作流沉淀

### 视频目标

把整个课程收束成可复用方法，让同学能迁移到自己的项目。

### 主要内容

- 主控会话和子会话的分工。
- `CLAUDE.md` 如何沉淀项目规则。
- 提示词模板如何复用。
- 什么时候新开会话。
- 如何把 POC 方法迁移到真实项目。
- 后续进阶：MCP、RAG、内部知识库、团队 skill、代码审查自动化。
