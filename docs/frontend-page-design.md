# 前端页面设计基线

## 主控验收结论

本前端页面设计通过主控窗口验收，可以作为后续 `poc/frontend` 代码实现、前后端联调和第 6 课录屏的设计基线。

通过理由：

- 页面定位是企业后台操作台，不是营销页。
- 完整覆盖“客户查询 -> 详情查看 -> 风险重算 -> 日志追踪”的最小演示闭环。
- API 调用严格来自 `docs/api-design.md`，没有新增后端契约。
- 保留 `masked` / `sample` 脱敏语义，适合银行研发安全边界讲解。
- 图表不依赖新增聚合接口，符合第一阶段边界。
- 方案清楚包含页面、路由、组件、状态、API 对接和录屏讲解顺序。

主控补充裁决：

- 第一阶段前端暂不实现客户新增 / 修改弹窗，避免课程主线发散。
- 前端第一阶段重点是客户列表、客户详情、风险重算、操作日志和仪表盘。
- 仪表盘可以基于 `GET /api/customers?pageNo=1&pageSize=100` 做 POC 级轻量聚合，但录制时必须说明这不是生产统计方案。
- 前端代码实现时不新增导出、批量导入、真实登录、权限和删除能力。

## 1. 设计目标与边界

目标：做一个适合银行研发培训录屏的企业后台操作台，完整演示“客户查询 -> 详情查看 -> 风险重算 -> 日志追踪”的最小闭环。

边界：

- 不做真实登录。
- 不做复杂权限。
- 不做导出、批量导入、删除。
- 不做真实风控模型。
- 不新增或改变后端 API 契约。
- 所有字段保留 `masked` / `sample` 语义。

## 2. 用户角色与流程

角色：

- 查询人员。
- 管理人员。
- 讲师 / 学员。

主流程：

1. 进入仪表盘。
2. 在客户列表筛选客户。
3. 查看客户详情。
4. 触发风险重算。
5. 刷新风险结果。
6. 到操作日志页查看 `requestId`、操作人、结果和失败样例。

## 3. 页面清单

- 仪表盘：总览、风险分布、客户状态统计、最近重算日志入口。
- 客户列表：筛选、分页表格、风险标签、详情入口。
- 客户详情：基础信息、账户列表、当前风险、历史风险记录。
- 风险重算弹窗：确认、重算中、结果展示、失败提示。
- 操作日志页：筛选、分页表格、成功 / 失败日志追踪。

## 4. 页面信息架构与核心组件

仪表盘：

- 顶部指标条。
- 风险等级分布图。
- 客户状态统计图。
- 最近操作表格。

客户列表：

- 筛选表单。
- 客户表格。
- 风险标签。
- 状态标签。
- 分页器。
- 查看详情按钮。

客户详情：

- 客户摘要区。
- 风险概览卡。
- 账户表格。
- 历史风险表格。
- 重算按钮。

风险重算弹窗：

- 客户号。
- 当前风险。
- 重算原因 `reasonSample`。
- 规则 `DEMO_RULE_V1`。
- 事务效果展示。

操作日志页：

- 请求号、操作人、操作类型、结果、时间范围筛选。
- 日志结果表格。

## 5. 视觉方向

采用克制、密集、清晰的企业后台风格：

- 左侧菜单 + 顶部面包屑 + 主内容区。
- 低风险绿色、中风险琥珀色、高风险红色。
- 普通信息用灰蓝，不做营销页式大渐变。
- 表格密度默认中等偏紧凑，适合录屏一屏展示更多字段。
- 图表低饱和、清晰图例、悬浮 tooltip 展示数量和占比。

实现时应避免过度模板感。可以在标题层级、卡片间距、风险状态条和图表配色上做出明确但克制的设计识别。

## 6. 路由设计

- `/dashboard`：仪表盘。
- `/customers`：客户列表。
- `/customers/:customerNo`：客户详情。
- `/operation-logs`：操作日志。
- 默认重定向到 `/dashboard`。

风险重算作为客户详情页内弹窗，不单独做路由。

## 7. API 对接清单

仪表盘：

- `GET /api/customers?pageNo=1&pageSize=100`：用于 POC 样例聚合风险分布和客户状态。
- `GET /api/operation-logs?pageNo=1&pageSize=5`：展示最近日志。

客户列表：

- `GET /api/customers`

客户详情：

- `GET /api/customers/{customerNo}`
- `GET /api/customers/{customerNo}/accounts`
- `GET /api/customers/{customerNo}/risk`
- `GET /api/customers/{customerNo}/risk-records`

风险重算：

- `POST /api/customers/{customerNo}/risk/recalculate`

操作日志页：

- `GET /api/operation-logs`

## 8. 状态设计

- 加载：表格骨架屏或 Element Plus loading。
- 空数据：显示“暂无样例数据”，不要暗示真实客户为空。
- 错误：展示统一响应的 `message`、`code`、`requestId`。
- 重算中：弹窗按钮 loading，禁止重复提交。
- 重算成功：展示新风险等级、分数、`transactionEffect` 三项结果。
- 重算失败：展示 `404001` / `409001` 等业务错误，并提示可到日志页按 `requestId` 查询。

## 10. ECharts 使用建议

优先做两个图：

- 风险等级分布饼图或环图。
- 客户状态柱状图。

客户详情页可做账户概览小图：按账户类型或账户状态统计，但数据只来自当前客户的账户列表。

不要为了图表新增后端聚合接口。第一阶段可在前端基于样例分页数据做轻量聚合，并在讲解中说明这是 POC 展示策略。

## 11. 安全与脱敏展示

- 界面字段名保留 `mobileMasked`、`idNoMasked`、`accountNoMasked`、`riskReasonSample`、`errorMessageSample` 等语义。
- 不展示数据库自增 `id`。
- 错误提示不展示堆栈、SQL、连接串。
- 录屏中只使用 `CUST10000*`、`SAMPLE-*`、`demo-admin`、`demo-query` 这类样例值。

## 12. 前端实现顺序建议

1. 搭 Vite + Vue 3 + TS + Element Plus 基础壳。
2. 做 API client、统一响应解包、错误处理和 `X-Operator` / `X-Request-Id`。
3. 做客户列表。
4. 做客户详情和账户 / 风险 / 历史记录。
5. 做风险重算弹窗。
6. 做操作日志页。
7. 做仪表盘和 ECharts 聚合展示。
8. 做手工验收和录屏路径打磨。

## 13. 课程录制讲解顺序

1. 先讲前端不改 API，只消费 `/api/...`。
2. 讲布局和页面拆分。
3. 演示客户列表筛选、详情聚合、多接口并行加载。
4. 重点讲风险重算：请求体、事务效果、页面刷新、日志追踪。
5. 讲脱敏字段、错误码、`requestId` 和第二阶段 Gateway 兼容。

## 14. 风险点与待确认

风险点：

- 仪表盘没有专用聚合 API，前端只能用样例分页数据聚合。
- 账户总览不能跨全量账户准确统计。
- 如果后端返回字段和契约漂移，前端表格和详情页会受影响。

待确认：

- 客户维护弹窗是否进入第一阶段前端实现。主控裁决：暂不进入第一阶段。
- 详情查询失败是否自动跳回列表页。建议保留错误页态，不自动跳转，方便录屏讲错误处理。
