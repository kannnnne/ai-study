# 案例库

## 案例选择原则

- 必须贴近日常研发，而不是炫技。
- 输入材料要小，方便录屏演示。
- 结果要可验证，例如测试通过、文档生成、调用链能对上。
- 尽量避开真实客户数据、生产配置、密钥和敏感代码。

## 推荐案例 1：读懂一个陌生 Java 项目

### 场景

新同学接手一个小服务，不知道入口、配置和核心业务在哪里。

### AI 任务

- 总结目录结构。
- 找入口类和核心 service。
- 梳理接口调用链。
- 输出优先阅读文件列表。

### 课程价值

这个案例最容易让同学有获得感，因为代码理解是每个项目都会遇到的痛点。

## 推荐案例 2：银行接口字段映射说明

### 场景

项目里有请求 DTO、响应 DTO、XML 或 JSON 报文，需要整理字段含义和映射关系。

### AI 任务

- 从 DTO 中提取字段。
- 生成字段表：字段名、类型、是否必填、来源、说明。
- 标记不确定字段，要求人工确认。

### 课程价值

非常贴合银行解决方案研发，适合展示 AI 对文档和接口梳理的效率提升。

## 推荐案例 3：新增一个小字段

### 场景

在查询响应中新增一个业务字段，需要修改 DTO、service、测试和文档。

### AI 任务

- 分析影响范围。
- 修改代码。
- 补充测试。
- 输出变更说明。

### 课程价值

展示完整开发闭环：分析、修改、验证、总结。

## 推荐案例 4：补单元测试

### 场景

已有服务方法逻辑不复杂，但缺少测试。

### AI 任务

- 提取测试矩阵。
- 生成测试代码。
- 根据报错迭代修复。

### 课程价值

投入产出比高，适合说服团队：AI 不只会聊天，也能提升工程质量。

## 推荐案例 5：日志排查

### 场景

给出一段异常日志和相关代码，让 AI 协助定位可能原因。

### AI 任务

- 从日志提取关键错误。
- 定位可能代码路径。
- 给出排查顺序。
- 提出最小修复或验证方式。

### 课程价值

适合强调 AI 的边界：AI 可以帮忙推理，但最终要靠日志、代码和测试验证。

## 推荐案例 6：代码审查

### 场景

准备一个包含隐患的 pull request 或 diff。

### AI 任务

- 按严重程度列出问题。
- 指出证据文件和代码位置。
- 区分确定问题和建议优化。

### 课程价值

让同学建立“AI 输出需要证据”的习惯。

## 可演示的银行研发问题清单

- 金额计算不能用浮点数。
- 日期和时区处理错误。
- 空字段导致空指针。
- 外部接口超时和重试策略。
- 响应码映射不完整。
- 日志打印敏感信息。
- DTO 字段名和接口文档不一致。
- 异常被吞掉，问题难排查。
- 单元测试只覆盖正常路径。
