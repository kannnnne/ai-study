# 第 0 课录屏材料子任务

## 子任务目标

产出第 0 课“课程导入和预期管理”的可录制材料，用作整套 AI 编程入门教程的样片开场。

本节课的核心不是演示复杂技术，而是让学员和领导理解：

- 为什么现在要学 AI 编程。
- 这套课解决什么问题。
- 课程会交付什么。
- 学员学完能做到什么。
- AI 工具的边界和安全要求是什么。
- 讲师定位是“有实际使用经验、能带大家入门和落地”，不是夸大成大模型算法专家。

## 输入基线

必须阅读：

- `README.md`
- `docs/course-blueprint.md`
- `docs/lesson-plan.md`
- `docs/deliverables-and-launch-plan.md`
- `docs/leadership-brief.md`
- `docs/ai-dev-workflow.md`
- `docs/fullstack-integration-acceptance.md`
- `docs/lesson-06-recording-material.md`

## 子窗口提示词

```text
你现在是 AI 教程项目的第 0 课录屏材料子窗口。请只负责整理第 0 课“课程导入和预期管理”的录制材料，不写代码，不改项目。

课程背景：
- 这是一套面向赞同科技银行解决方案研发同学的 AI 编程入门教程。
- 教程以视频为主、文字为辅。
- 课程主线是用 Claude Code + DeepSeek 从 0 搭建一个最小可运行 Java 微服务 POC。
- 学员大多是有研发经验但 AI 编程经验较少的技术同学。
- 当前项目已经完成后端、前端和前后端联调闭环，并已有第 6 课录屏材料。

请先阅读：
- README.md
- docs/course-blueprint.md
- docs/lesson-plan.md
- docs/deliverables-and-launch-plan.md
- docs/leadership-brief.md
- docs/ai-dev-workflow.md
- docs/fullstack-integration-acceptance.md
- docs/lesson-06-recording-material.md

输出要求：
1. 视频标题和一句话目标。
2. 本节课面向学员和领导分别要解决什么问题。
3. 预计时长和章节划分，建议 12 到 18 分钟。
4. 录制前准备清单。
5. 开场 1 分钟口播稿，要求自然、可信、不夸大。
6. 完整半口语口播稿。
7. 建议展示哪些材料或画面：
   - 课程总目标
   - POC 最终效果
   - 课程交付物
   - AI 研发工作流
   - 主控会话和子会话分工
   - 安全和脱敏边界
8. 领导视角表达：如何说明这套课对团队、项目和管理推广的价值。
9. 学员视角表达：如何说明学完能做什么、不能做什么。
10. 讲师自我定位表达：既体现能力，又不夸大成算法专家。
11. 本节课必须讲清的边界：
    - AI 是辅助研发，不是替代责任
    - 本课程不是大模型算法课
    - 不上传真实敏感数据
    - AI 输出必须测试和人工验收
12. 可放到讲义里的精简版文字材料。
13. 课后行动建议。
14. 失败兜底方案：如果领导问 ROI、如果同学问没有代理怎么办、如果有人质疑 AI 会不会乱写代码、如果环境装不起来怎么办。
15. 主控窗口验收清单。

风格要求：
- 像真实技术负责人给同事做内部培训开场。
- 通俗，但不要鸡血。
- 不要说 AI 会替代程序员。
- 不要把讲师包装成大模型底层专家。
- 要让领导觉得这件事有组织价值，也让学员觉得自己能上手。
- 重点体现：我们不是只教工具，而是沉淀一套安全、可控、可复制的 AI 辅助研发工作流。

最后请给主控窗口一段简短汇报，说明这份材料是否可以用于录制第 0 课样片。
```

## 主控验收重点

- 是否适合做整套课程开场。
- 是否能同时打动领导和技术学员。
- 是否自然说明讲师能力和边界。
- 是否讲清课程交付物和学习路径。
- 是否能和第 6 课联调样片形成“价值说明 + 实际效果”的组合。
- 是否没有夸大 AI、没有弱化安全和人工验收责任。
