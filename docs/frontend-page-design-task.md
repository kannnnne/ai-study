# 前端页面设计子任务

## 子任务目标

基于后端 API 契约和已实现的第一阶段后端，设计 Vue 3 企业后台页面方案。当前子任务只做前端设计和联调计划，不创建代码项目，不写 Vue 代码。

## 输入基线

实现前必须阅读：

- `docs/poc-architecture.md`
- `docs/api-design.md`
- `docs/backend-code-implementation-acceptance.md`
- `docs/backend-implementation-plan.md`

## 设计方向

这是面向银行研发团队的 AI POC 前端，不是营销页。页面应该像一个克制、专业、可信的企业级操作台。

视觉原则：

- 清晰、稳重、可演示。
- 信息密度适中，适合录屏讲解。
- 避免炫技、复杂动效、营销大图。
- 避免默认 AI 味道和泛泛模板感。
- 风险等级要有清晰视觉层级，但不要过度夸张。

技术栈裁决：

- Vue 3
- TypeScript
- Vite
- Element Plus
- ECharts

请使用 frontend-design skill 的思路：先理解用户、场景、信息架构、视觉方向和组件结构，再给出设计方案。这个项目是企业后台操作台，不是营销页。

## 主控验收重点

- 是否匹配企业后台，而不是营销页。
- 是否完整覆盖客户列表、详情、风险重算、日志查看。
- 是否严格对齐 `docs/api-design.md`。
- 是否明确加载、错误、空状态和重算反馈。
- 是否保留脱敏字段的安全提醒。
- 是否适合录制第 6 课“前端页面和接口联调”。
